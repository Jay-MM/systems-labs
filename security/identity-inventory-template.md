# Identity Inventory Template

Do not store passwords, recovery codes, MFA codes, private keys, or account members in this file.

## Purpose

This template helps map critical accounts and check wether basic identity protections are in place.

## Critical Accounts

- Main email
- Backup email
- Password manager
- Bank / financial accounts
- Phone carrier
- Apple / Google account
- GitHub
- LinkedIn
- School account
- Cloud storage
- Social media

## Account Review Template

For each critical account, check:

Account ype:

MFA enabled?:yes / no / unsure
Unique password: yes / no / unsure
Recovery options updated: yes / no / unsure
Old sessions reviewed: yes / no / unsure
Security alerts enabled: yes / no / unsure

Notes:

## Priority Order

1. Main email
2. Password manager
3. Banking / finance
4. Phone carrier
5. Apple / Google account
6. GitHub / LinkedIn / school
7. Cloud storage
8. Social media

## Security Rules

- Never store passwords in plain text.
- Never commit recovery codes to GitHub.
- Never commit MFA backup codes to GitHub.
- Never commit private keys.
- Keep sensitive account details in a password manager or secure private storage.
- Public repositories should only contain templates, checklists, and sanitized examples.
